# Auto-QC Agent 上下文

> 外呼通话文本质检 Skill — 基于 Claude Code 的大规模对话质检系统。

## 项目概述

为 Claude Code 构建的质检 skill，用于外呼通话对话文本的自动化质量检测。支持 1w-5w 条对话数据的大规模质检，输出合规检测结果和归因分析报告。

## 核心架构：统一框架，两套规则

质检和归因是同一件事——给对话打标，只是标签不同。框架（身体）完全一致，区别在于注入的规则包（灵魂）。

```
Supervisor Agent → 5x Worker Agent 并发 → Python I/O 层 → 规则层 + 数据层
         ↓ 两次运行，不同规则包
  通道一：合规检测（用户提供 rules.md）
  通道二：归因分析（内置 attribution-rules.md）
```

### 职责分工

| 谁 | 负责什么 |
|---|---|
| **Python** | Excel 读取、批次拆分、对话预处理、报告输出、进度管理 |
| **LLM (Sub-agent)** | 规则解析、逐批打标（理解对话 + 判断 + 输出）、交叉验证 |

**核心约束**：质检判断必须由 LLM 完成，禁止用 Python 代码替代 LLM 做质检判断。

## 运行模式

| 命令 | 行为 |
|---|---|
| `/auto-qc --data <路径> --rules <路径>` | 默认全量：合规检测 + 归因分析 |
| `/auto-qc --data <路径> --rules <路径> --no-attribution` | 仅合规检测 |
| `/auto-qc --data <路径> --attribution-only` | 仅归因分析（使用内置归因规则） |

## 批次参数

- **每批 100 条对话**，每次并发 5 个 Worker Agent
- 1w 条数据 = 100 批，约 20 轮并发
- Worker 防偷懒约束：逐条输出、规则遍历清单、抽检内嵌

## Excel 列识别

按列名关键字匹配，不依赖固定列序：

| 列 | 匹配关键词 |
|---|---|
| ID | `id`, `通话ID`, `call_id` |
| 时间 | `时间`, `通话时间`, `call_time` |
| 对话 | `对话`, `对话文本`, `conversation` |
| 意向 | `意向`, `意向结果`, `intent_result` |

## 规则文件格式

```markdown
## R01: 规则名称
**严重程度**: 高/中/低
**发现次数**: 数字
**描述**: ...
**检测逻辑**: ...
**典型案例**:
- 案例1
- 案例2
```

## 输出报告

| Sheet | 内容 |
|---|---|
| Sheet 1 | 合规检测结果（id / 时间 / 意向结果 / 违规规则 / 问题类型 / 危害程度 / 证据片段 / 改进建议） |
| Sheet 2 | 归因分析（意向结果 / 归因类别 / 占比 / 典型案例 / 改进建议） |
| Sheet 3 | 统计概览（规则明细表：规则ID/名称/命中次数/占比 + 合计行） |

## 失败重试策略

- Worker 失败/超时/输出格式异常 → 自动重试，最多 3 次
- 3 次都失败的批次 → 记录到 `failed_batches.json`，不阻塞整体流程
- JSON 容错：先用 `json_repair` 修复，再 `json.loads`，仍失败则重试

- `json-repair==0.42.0` — 不规范 JSON 修复

## 断点续跑

`progress.json` 实时记录状态。启动时自动检测，存在未完成任务时提示用户继续或重头开始。

## 关键决策记录

1. Prompt 驱动 + 少量 Python I/O，质检判断全部由 Claude sub-agent 完成
2. 归因分析是内置能力，默认开启，无需指定规则文件
3. 合规检测规则必须由用户提供，框架不做假设
4. 按意向结果类别分组输出归因报告（B/C/E/F/I）
5. Skill 可移植：zip 解压到 `~/.claude/skills/auto-qc/` 即可使用

## 端到端测试验证结果

使用真实 500 条数据完成全流程：

**合规检测**：违规率 80.6%（403/500），最高频违规 R10(结束话术乱码) 281次、R02(对话未正常结束) 128次、R05(回避用户问题) 103次。

**归因分析**：370 条非 A 意向对话。I(开场白挂断) 主要归因：用户无实质性回应(50.3%)、AI质量缺陷(49.7%)；F(无意向) 主要归因：AI质量缺陷(97.7%)、用户明确拒绝(91.5%)；B(不确定) 主要归因：AI质量缺陷(79.2%)、用户询问细节但意向不明(45.5%)。
